1. 外观模式：纯对不同情境包装不同功能函数，可以理解成封装多个功能函数。
2. 适配器模式：对函数的参数进行包装，或者重写函数，以达到适配其他框架的功能。
3. 代理模式：对于已经有的接口，我们无法修改，如果想在接口上添加功能，就可以外面包装一层代理。代理的参数和返回应该与接口一致，这样使用接口时，就很容易替换成代理接口。与装饰者不同的是，我们不对原函数的返回值进行包装，而是提供和原函数一模一样出入参数的函数，去调用原函数。
4. 装饰者模式：对函数的返回值进行包装，形成一个新的函数，新的函数包含原函数的所有功能，又做了其他扩展。
5. 桥接模式：对函数的具体实现进行包装，可以理解成封装一个功能函数。
6. 组合模式：对函数进行集合包装，当执行包装函数的某些方法时，就会执行被包装函数的同名方法。
7. 享元模式：由于对象的数据类型相同，在很多情况下，可以共用同一个实例化对象完成需求。
